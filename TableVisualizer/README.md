# TableVisualizer 遊戲數值表視覺化工具

TableVisualizer 是一個跨平台（macOS / Windows）的桌面 GUI 工具，用 **PyQt6** 打造，
專門用來**視覺化遊戲數值表（Excel `.xlsm` 檔）之間的關聯**，並讓你直接在工具內
瀏覽、搜尋、編輯儲存格，再一鍵跳回 Excel。

> 資料流：Excel 檔 → 掃描結構 → 自動建立關聯圖 → GUI 互動檢視 / 編輯

## 安裝與執行

```bash
# 1) 建立虛擬環境並安裝相依套件
python3 -m venv venv && source venv/bin/activate && pip install -r requirements.txt

# 2) 執行（macOS）
./start_visualizer.command          # 或： python3 table_visualizer_gui.py
```

相依套件：`openpyxl`、`pandas`、`PyQt6`、`networkx`、`matplotlib`。

打包成獨立應用程式：

```bash
./build_mac.command                 # macOS → 產出 dist/mac
./build_pc.bat                      # Windows
```

## 功能總覽

### 1. 工作目錄管理

- **啟動時選擇 Excel 目錄**：第一次開啟會跳出對話框，請你指定放置 Excel 數值表的資料夾，並即時驗證目錄是否有效。
- **記憶上次目錄**：選過的目錄會被記住，下次開啟自動沿用（驗證有效時）。
- **隨時切換目錄**：頂部「變更工作目錄」按鈕可改指向另一組 Excel，切換後自動重新掃描與載入。
- JSON 設定檔固定存放在程式目錄；Excel 檔則在你選定的工作目錄，兩者分離互不干擾。

### 2. 自動掃描與關聯分析

- **一鍵重新掃描**：頂部「Rescan Data」會重新掃描整個工作目錄，並以進度條顯示「掃描檔案 → 分析關聯 → 載入資料」三階段。開啟 App 時也會自動靜默刷新一次。
- **三種自動關聯偵測**：
  1. **命名階層（naming_hierarchy）**：`TableName_子表` 自動關聯回同檔的父工作表。
  2. **欄位參照（string_reference）**：某張表的欄位名稱若包含另一張工作表的名稱（最長匹配優先），即建立關聯，並標示是哪個欄位觸發。
  3. **數值參照（value_reference）**：欄位的樣本值若包含其他工作表名稱，同樣建立關聯。
- **穩定節點 ID**：節點以 `Table.Sheet` 全名的雜湊值為 ID，即使新增 / 刪除工作表後重新掃描，既有關聯也不會錯位。
- 以 `#` 開頭的工作表（列舉 / 參考表）會自動被過濾，不顯示為節點。

### 3. 主瀏覽器（左上）

- **分組卡片瀏覽**：所有工作表以所屬 Excel（Table）分組，依表格顏色區分，垂直捲動瀏覽。
- **A–Z 字首索引**：左側索引條可快速跳到指定字首的分組，捲動時索引也會同步高亮。
- **點擊卡片** → 選中該工作表（會在 Relation View 顯示關聯、在右側面板顯示內容）。
- **雙擊卡片** → 直接用 Excel 開啟該工作表。
- **中鍵卡片** → 釘選 / 取消釘選（見[釘選](#6-釘選與多表切換)）。
- **點擊分組標題** → 跳出取色盤，自訂該 Table 的代表色（即時套用並存檔）。
- **搜尋高亮 / 篩選**：頂部搜尋框可即時篩選、高亮符合的卡片。

### 4. 關聯示意視窗 Relation View（左下）

- 以選中的節點為中心，左側列出**上游（指向它）**、右側列出**下游（它指向的）**關聯節點，箭頭標示方向，顏色與主瀏覽器同步。
- **中鍵拖曳平移**、**滾輪縮放**自由檢視。
- **點擊關聯節點** → 在右側面板顯示該節點內容；**雙擊** → 用 Excel 開啟。
- **中鍵節點** → 切換釘選（不切換預覽、不導航）。
- **缺失節點（Ghost）提示**：若某關聯指向的工作表已被刪除，會以紅色虛線框 + 「(Violated)」標示，並提供 X 按鈕移除，協助你發現失效參照。

### 5. 即時預覽 / 編輯面板（右側）

- **完整內容預覽**：選中任一工作表即顯示其完整內容，**還原 Excel 的原始儲存格填色**（含主題色 + tint 轉 RGB），凍結標題列並以粗黑線分隔。
- **直接編輯儲存格**：可在面板內修改值。
- **Save to Excel**：把修改寫回 Excel（Windows 走 COM、macOS 走 AppleScript、其他平台走 openpyxl，並以 `keep_vba=True` 保護 `.xlsm` 巨集）。
- **Open in Excel**：一鍵以 Excel 開啟目前工作表。
- **右下角縮放拉條**（50%–300%）：像 Excel 一樣調整預覽表格大小。
- **全介面切換（⛶ / Tab）**：把右側面板放大至整個視窗，再按一次還原。
- **未儲存提醒**：若有未存編輯卻要切換到別張表，會跳出「保存 / 不保存 / 取消」確認，避免誤失修改。

### 6. 釘選與多表切換

- 把常看的工作表**釘選**起來，會在右側面板上方形成一排「子表格方塊（chips）」。
- **拖曳 chips 可重新排序**釘選清單。
- **Ctrl+Tab / Ctrl+Shift+Tab**（macOS 為實體 Control）在釘選的表格間快速循環切換預覽，與切換瀏覽器分頁的手感一致。
- 釘選狀態僅存於記憶體，關閉 App 即解除。

### 7. 表格內容搜尋

右側面板提供跨表格的內容搜尋：

- **三種搜尋範圍**：目前瀏覽表格 / 所有釘選表格 / 所有表格。
- **Enter 找下一筆、Shift+Enter 找上一筆**，並顯示符合筆數與目前位置，定位後自動高亮對應儲存格。

### 8. 手動關聯編輯

- 自動偵測之外，你可以**手動新增 / 刪除關聯**：在 Relation View 編輯模式下，節點旁的綠色「＋」可新增關聯（選擇方向與目標表）、連線中點的紅色「✕」可刪除。
- 手動關聯獨立記錄於 `user_relations.json`，**重新掃描也不會被覆蓋**（新增的會保留、被你刪掉的會持續隱藏）。

### 9. 設定與外觀

「Settings」對話框提供：

- **全域字體大小**（即時預覽，8–24）。
- **預覽欄寬上限**（字數截斷門檻，100–2000）：控制右側預覽過長內容的截斷。
- 深色主題介面，跨平台一致。

### 10. 狀態持久化

關閉時自動保存到 `gui_config.json`，下次開啟還原：

- 視窗大小 / 位置 / 是否最大化
- 三窗格分隔線比例
- 各 Table 的自訂顏色
- 全域字體大小、預覽縮放比例、預覽欄寬上限
- 工作目錄

## 鍵盤與滑鼠操作速查

| 操作 | 結果 |
|------|------|
| 點擊卡片 / 節點 | 選中並顯示關聯與內容 |
| 雙擊卡片 / 節點 | 用 Excel 開啟該工作表 |
| 中鍵卡片 / 節點 | 釘選 / 取消釘選 |
| 中鍵拖曳（Relation View） | 平移畫面 |
| 滾輪（Relation View） | 縮放 |
| 點擊分組標題 | 變更該 Table 顏色 |
| 拖曳釘選方塊 | 重新排序釘選 |
| `~` | 快速聚焦頂部搜尋框 |

## 設定檔說明

| 檔案 | 用途 |
|------|------|
| `excel_structure.json` | Excel 結構掃描結果（工作表、欄位、列數、樣本值、填色）。 |
| `relationship_graph.json` | 節點與連線資料（GUI 載入的關聯圖）。 |
| `gui_config.json` | 使用者 GUI 設定與狀態（座標、顏色、字體、版面、工作目錄等）。 |
| `user_relations.json` | 使用者手動新增 / 刪除的關聯，重新掃描不受影響。 |

### 程式架構

| 模組 | 職責 |
|------|------|
| `table_visualizer_gui.py` | 主程式與所有 PyQt6 UI，入口 `main()`，核心類別 `MainWindow`。 |
| `analyze_excel_structure.py` | 掃描 Excel 目錄，產生 `excel_structure.json`。 |
| `relationship_analyzer.py` | 偵測命名 / 欄位 / 數值參照關聯，輸出 `relationship_graph.json`。 |
| `excel_handler.py` | 跨平台讀寫 Excel、解析儲存格填色、跳轉開啟工作表。 |
| `config_manager.py` | 讀寫 `gui_config.json`。 |
