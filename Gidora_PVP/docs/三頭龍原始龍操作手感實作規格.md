# 三頭龍原始龍操作手感實作規格

規格件立者: lin neil  
標籤: Gidora / Original Dragon / Controls  
狀態: 工程實作規格  

## 實作範圍

本文件只定義「原始龍」型態的操作與手感，供 Unity 正式專案實作使用。HTML 版本僅作為手感參考，數值以目前測試版已調整到位的行為為準。

- 適用：P1/P2/P3 三顆頭、P4 尾巴。
- 包含：移動、Melee、蓄力 Melee、集氣 / 光束炮。
- 不包含：Buff、子彈龍、噴火龍、藤蔓、飛天墜擊、其他型態替換技。

## 核心互斥規則

### 行為優先度

每個操作部位各自判斷輸入貢獻，優先度如下：

1. **攻擊 = 集氣**
2. **移動**

攻擊與集氣同級，採「後蓋前」：

- 先按 Melee、再按集氣：改以集氣為準，取消尚未出手的 Melee 預備 / 蓄力。
- 先按集氣、再按 Melee：改以 Melee 為準，該部位停止貢獻集氣。
- 同一幀無法判斷先後時，維持目前行為狀態，避免來回抖動。
- 移動永遠不會覆蓋攻擊或集氣；只要該部位正在攻擊或集氣，該部位的移動輸入不參與位移與轉向。

### 行為鎖定

- **集氣中**：該部位不能移動，不能同時維持 Melee；若較新的 Melee 輸入生效，先結束該部位集氣貢獻再進入 Melee。
- **Melee 中**：該部位不能移動，不能同時貢獻集氣；若較新的集氣輸入生效，先取消尚未出手的 Melee 預備 / 蓄力再進入集氣。
- **移動中按 Melee / 集氣**：該部位移動輸入立即歸零，不再提供速度或轉向。
- **光束炮施放中**：所有 Melee 取消；原始龍光束炮期間不接受位移輸入。

## 移動邏輯

### 輸入合成

每幀收集四個部位的移動向量，排除正在攻擊或集氣的部位。

- 單一部位輸入長度上限為 1。
- 有效移動向量加總為 `MoveSum`。
- 有效轉向向量加總為 `TurnSum`。
- `MoveSum` 只代表「是否有推動龍前進」；實際加速方向永遠沿龍身目前面向。
- `TurnSum` 代表龍身想轉向的目標方位。

### 車輛式移動手感

三頭龍不直接把速度改成輸入方向，而是模擬高抓地力車輛：

- 有有效移動輸入時，龍身先朝 `TurnSum` 的方向轉向。
- 速度方向以高抓地力快速貼回龍身面向，避免側滑或甩尾。
- 加速度永遠加在龍身 forward。
- 無有效移動輸入時，以摩擦力讓速度降到 0。

### 轉向降速

根據龍身當前面向與 `TurnSum` 目標方向的角度差，套用最高速懲罰：

- `0° - 30°`：最高速 100%。
- `30° - 180°`：線性衰減。
- `180°`：最高速 50%。

此規則用來保留巨獸重量感；大角度掉頭會先慢下來，再重新加速。

### 同心協力加速

若至少兩個有效移動方向相近，視為同心協力移動：

- 任兩個有效方向 dot > `0.707`（約 45 度內）時觸發。
- 最高速倍率：`1.5x`。
- 轉向速度倍率：`1.5x`。

### 移動參數

| 參數 | 數值 | 說明 |
|---|---:|---|
| `MaxSpeed` | `7` | 基礎最高速度 |
| `Acceleration` | `7` | 每秒加速度 |
| `Friction` | `10` | 無輸入時減速 |
| `Traction` | `100` | 速度方向貼齊面向速度 |
| `RotationSpeed` | `3.1` | 基礎轉向速度 |
| `TeamworkSpeedMultiplier` | `1.5` | 同心協力最高速倍率 |
| `TeamworkTurnMultiplier` | `1.5` | 同心協力轉向倍率 |

## P1/P2/P3 頭部 Melee

### 基本流程

P1/P2/P3 使用同一套頭槌流程：

1. 按下 Melee：進入 `Charging`，頭部往後仰，開始計算按住時間。
2. 按住期間：可用該部位移動方向 / 瞄準方向調整落點。
3. 放開 Melee：依按住時間決定輕擊或蓄力重擊。
4. 進入 `Strike`：頭部向前打出。
5. 命中幀後進入 `Recovery`。
6. 後搖結束回到 `Idle`。

### 輕擊

放開時按住時間未達 `ChargeTime` 即為輕擊。

- 傷害：`MeleeDamage = 16`。
- 半徑：`AttackRange = 2`。
- 擊退：`MeleeKnockback = 10`。
- 命中位置：
  - P1：龍身 forward 左偏 `-0.6 rad`，距離約 `3.0`。
  - P2：龍身 forward 右偏 `+0.6 rad`，距離約 `3.0`。
  - P3：龍身 forward，距離約 `3.2`。
- 打擊判定為短時間圓形範圍，同一目標每次攻擊只命中一次。

### 蓄力重擊

按住達 `ChargeTime` 後放開即為蓄力重擊。

- 蓄滿時間：`ChargeTime = 0.4s`。
- 傷害倍率：`HeavyDamageScale = 3.0`，即 `16 * 3 = 48`。
- 範圍倍率：`HeavyRadiusScale = 1.4`，即半徑 `2.8`。
- 額外失衡：命中時額外給予傷害 `50%` 的失衡值。
- 落點距離：以龍身為中心固定 `4.2`。
- 落點顯示：蓄滿後顯示圓形落點提示。

### 頭部瞄準限制

蓄力期間瞄準方向以各頭預設方向為中心限制扇形角度：

| 部位 | 預設 yaw | 可偏轉半角 |
|---|---:|---:|
| P1 | `-0.6 rad` | `±60°` |
| P2 | `+0.6 rad` | `±60°` |
| P3 | `0 rad` | `±60°` |

若該部位沒有瞄準輸入，維持預設方向。

### 頭部攻擊時間

| 階段 | 數值 | 說明 |
|---|---:|---|
| `WindupTime` | `0.12s` | 按下後往後仰的最短前搖 |
| `StrikeTime` | `0.16s` | 從後仰打到前方 |
| `ImpactTiming` | `Strike` 進度 `65%` | 觸發傷害判定 |
| `RecoveryTime` | `0.4s` | 輕擊後搖 |
| `HeavyRecoveryTime` | `0.9s` | 蓄力重擊後搖 |

若玩家在 `WindupTime` 內就放開 Melee，仍需等前搖完成後才出手。

### 命中反作用力

頭部 Melee 命中下列目標時，攻擊者會受到反方向推力：

- 命中敵人 / 敵方三頭龍：`HeadRecoilForce = 4`。
- 命中實體建築 / 障礙物：`BlockRecoilForce = 5`。
- 尾巴攻擊不套用頭部反作用力。

## P4 尾巴 Melee

P4 使用同一顆 Melee，但行為與頭部不同。

### 尾巴按住狀態

1. 按下 Melee：進入 `TailCharging`，尾巴往右側預備。
2. 按住期間計算時間。
3. 放開時依按住時間決定輕掃或蓄力橫掃。

### 尾巴輕掃

放開時按住時間未達 `TailChargeTime` 即為輕掃。

- 傷害：`16`。
- 半徑：`2`。
- 命中位置：龍身後方，沿 `-forward` 約 `3.0`。
- 擊退：`TailMeleeKnockback = 15`。
- 後搖：`RecoveryTime = 0.4s`。
- 冷卻：`Cooldown = 0.6s`。

尾巴輕掃同時提供起步推進：

- 沿龍身 forward 直接注入主速度。
- 推進量：`TailLightDashForce = 9`。
- 可短暫超過當前最高速，超速額度 `TailLightDashOverspeed = 6`。
- 超速額度每秒以 `TailLightDashReturnRate = 10` 回正。

此推進是加到角色主速度上，不是單獨位移或擊退假動作。

### 尾巴蓄力橫掃

按住達 `TailChargeTime` 後放開即為蓄力橫掃。

- 蓄滿時間：`TailChargeTime = 0.4s`。
- 橫掃時間：`TailSweepDuration = 0.45s`。
- 龍身旋轉：橫掃期間龍身平滑旋轉 `180°`。
- 傷害觸發：橫掃進度 `40%` 時觸發一次全圓範圍傷害。
- 範圍：`TailSweepRadius = 5`。
- 傷害：`MeleeDamage * HeavyDamageScale * TailSweepDamageScale`，即 `16 * 3.0 * 1.7 = 81.6`。
- 擊退：`TailMeleeKnockback = 15`。
- 額外失衡：重擊額外失衡，尾巴橫掃需比頭部重擊更容易打出硬直。
- 後搖：`HeavyRecoveryTime = 0.9s`。

橫掃結束後龍身面向已反轉，接著進入短暫強制前進鎖：

- 持續：`TailSweepForwardLockDuration = 0.25s`。
- 期間只要任一部位提供移動輸入，輸入方向一律視為「沿新面向 forward 前進」。
- 輸入方向只代表「有沒有推」，不改變轉向目標。

## 集氣與光束炮

### 集氣規則

集氣按鍵由四個部位各自持有。只有符合下列條件的部位才會貢獻集氣：

- 該部位正在按住集氣。
- 該部位沒有正在 Melee。
- 光束炮處於 `Idle`。
- 光束炮不在冷卻中。

集氣值未增加時，以衰減速度下降。

| 同時集氣人數 | 每秒集氣值 |
|---:|---:|
| 0 | `-25` |
| 1 | `50` |
| 2 | `150` |
| 3 | `250` |
| 4 | `500` |

- 最大集氣值：`200`。
- 集滿後立即取消所有 Melee，進入光束炮。
- 冷卻期間按住集氣不增加能量，集氣值仍會衰減。

### 光束炮流程

原始龍集滿後施放光束炮：

1. `PreFire`：`0.5s`，蓄滿後發射前延遲。
2. `Firing`：`4.0s`，持續光束。
3. `PostFire`：`0.6s`，發射後收招。
4. 進入冷卻：`15.0s`。

### 光束炮操作限制

- `PreFire` / `PostFire`：不接受移動與 Melee。
- `Firing`：不接受位移，既有速度快速衰減至 0。
- `Firing` 期間可保留少量轉向手感；若實作，轉向速度建議為平時 `35%`。
- 施放期間若進入跌倒 / 失衡中斷，光束炮立即結束，集氣值歸 0。

### 光束炮命中

| 參數 | 數值 | 說明 |
|---|---:|---|
| `Range` | `25` | 最大射程 |
| `Width` | `0.8` | 光束寬度 / 判定半徑 |
| `TickInterval` | `0.1s` | 傷害 tick 間隔 |
| `DamagePerTick` | `8 * 1.6 = 12.8` | 每 tick 傷害 |
| `TickKnockback` | `2.5` | 每 tick 微擊退 |

光束會被實體目標截斷，傷害只作用在目前光束長度內。

## 狀態機建議

每個部位至少需要下列狀態：

- `Idle`
- `Charging`
- `Strike`
- `Recovery`
- `TailCharging`（P4）
- `TailSweep`（P4）
- `ComboCharging`

整隻龍至少需要下列全身狀態：

- `Normal`
- `BeamPreFire`
- `BeamFiring`
- `BeamPostFire`
- `Staggered / Downed`

移動系統每幀只讀取已通過互斥規則的有效移動輸入；不要讓移動系統自行覆蓋攻擊或集氣狀態。

## 實作檢查項

- 攻擊、集氣、移動必須互斥，且攻擊 / 集氣同級採後蓋前。
- P1/P2/P3 蓄力滿後才顯示落點提示。
- 頭部重擊落點固定距離 `4.2`，不是自由拖曳到任意距離。
- 大角度轉向會降最高速，但不應出現賽車甩尾。
- 尾巴輕掃會注入主速度並可短暫超速。
- 尾巴重擊會讓龍身 180 度轉向，結束後短暫強制新面向前進。
- 光束炮集氣人數越多越快，滿值自動施放。
- 光束炮施放時取消所有 Melee，且原始龍光束炮期間不能移動。
