# 失衡與跌倒機制實作規格

規格件立者: lin neil  
標籤: Stagger / Downed / Combat Feedback  
狀態: 工程實作規格  

## 實作範圍

本文件定義三頭龍與敵人的失衡條、跌倒、站起、受傷倍率、擊退與技能中斷規則。正式 Unity 專案需將此機制視為戰鬥核心狀態，不只是 UI 顯示。

- 適用：玩家三頭龍、敵人、PVP 敵方三頭龍。
- 包含：失衡累積、失衡衰退、跌倒、站起、受傷加成、攻擊 / 光束中斷、敵人倒地停手。
- 不包含：Buff 平衡設計；但會列出目前需保留的 Buff 介面掛點。

## 核心概念

每個可失衡單位都持有一條 `StaggerValue`。

- 受到有效傷害時，依規則累積失衡值。
- `StaggerValue >= Threshold` 時進入跌倒。
- 進入跌倒時，`StaggerValue` 歸 0。
- 失衡值未被刷新時，會隨時間下降。
- 跌倒期間仍可受傷，但不再累積新的失衡。

## 參數總表

### 玩家三頭龍

| 參數 | 數值 | 說明 |
|---|---:|---|
| `PlayerThreshold` | `450` | 玩家失衡條滿值 |
| `PlayerWindow` | `0s` | 玩家受傷後延遲多久才開始衰退 |
| `PlayerRecoveryRate` | `25 / s` | 玩家失衡值每秒衰退量 |
| `PlayerFallDuration` | `3.1s` | 玩家跌倒不可操作時間 |
| `PlayerStandUpDuration` | `0.5s` | 玩家站起動畫 / 不可操作時間 |
| `StaggeredDamageBonusPct` | `30%` | 玩家在失衡累積、跌倒或站起時受到額外傷害 |

### 敵人

| 參數 | 數值 | 說明 |
|---|---:|---|
| `EnemyThreshold` | `60` | 敵人失衡條滿值 |
| `EnemyWindow` | `1.0s` | 敵人受傷後延遲多久才開始衰退 |
| `EnemyRecoveryRate` | `85 / s` | 敵人失衡值每秒衰退量 |
| `EnemyFallDuration` | `1.2s` | 敵人跌倒不可行動時間 |

## 玩家三頭龍失衡流程

### 受到傷害

玩家三頭龍受到傷害時，同時處理 HP、失衡與擊退。

基本流程：

1. 若死亡或處於無敵防護，忽略本次傷害。
2. 計算實際 HP 傷害 `FinalDamage`。
3. 計算失衡傷害 `StaggerDamage`。
4. 套用失衡累積。
5. 套用擊退速度。
6. 若 HP 歸 0，進入死亡。

### HP 傷害與失衡傷害

玩家三頭龍的失衡傷害預設等於本次原始傷害，但不是所有防禦都會降低失衡。

- 正面 / 背面受傷倍率會同時影響 HP 與失衡。
- 站定防護類倍率會同時影響 HP 與失衡。
- 防禦提升只降低 HP 傷害，不降低失衡。
- 若處於「已失衡累積中、跌倒中、站起中」，HP 與失衡傷害都增加 `30%`。
- PVP 或特殊攻擊可傳入額外失衡倍率，例如破勢類效果。

### 玩家失衡累積

玩家可累積失衡時：

- `FallTimer <= 0`
- `StandUpTimer <= 0`
- `StaggerDamage > 0`

累積公式：

```text
StaggerValue += StaggerDamage * IncomingStaggerMultiplier
StaggerValue = min(StaggerValue, PlayerThreshold)
```

若累積後達到 `450`，立即進入跌倒。

### 玩家失衡衰退

玩家每幀更新失衡衰退：

```text
if StaggerWindowTimer > 0:
    StaggerWindowTimer -= dt
else:
    StaggerValue -= PlayerRecoveryRate * RecoveryMultiplier * dt
```

目前 `PlayerWindow = 0`，所以玩家受傷後不保留延遲窗口，下一幀即可開始衰退。

## 玩家跌倒

### 進入跌倒瞬間

玩家失衡條滿時，需立即執行：

- `FallTimer = 3.1s`
- `StaggerValue = 0`
- 所有部位攻擊狀態強制回 `Idle`
- 清除所有攻擊放開佇列
- 隱藏 Melee 落點提示
- 隱藏持續攻擊提示
- 清除或暫停會持續生效的 Buff 效果，持續時間覆蓋跌倒與站起總長
- 角色速度與擊退速度各乘 `0.2`
- 播放跌倒衝擊特效

### 光束炮中斷

若玩家在光束炮 `PreFire` 或 `Firing` 期間進入跌倒：

- `BeamCharge = 0`
- `BeamPhase = Idle`
- 關閉光束視覺
- 關閉特殊組合技視覺
- 清除一次性組合技倍率旗標
- 若角色高度高於地面，立即拉回地面

`PostFire` 不需額外中斷，因為已經是收招階段。

### 跌倒期間

`FallTimer > 0` 時：

- 不處理玩家輸入。
- 不允許移動。
- 不允許 Melee。
- 不允許集氣。
- 不允許施放光束炮。
- 不累積新的失衡。
- 仍可受到 HP 傷害。
- 受到傷害仍套用跌倒增傷 `30%`。
- 擊退速度每幀歸零，避免倒地時被連續推走。
- 主速度快速衰減，避免跌倒滑行。
- 角色往前 / 側向倒下，視覺角度朝倒地姿態內插。

跌倒計時結束後，進入站起。

## 玩家站起

### 站起流程

跌倒時間結束後：

- `StandUpTimer = 0.5s`
- 記錄目前倒地角度作為站起起點。
- 角色角度平滑內插回正常站姿。
- 主速度快速衰減。
- 擊退速度保持 0。
- 不處理玩家輸入。
- 不累積新的失衡。
- 仍可受到 HP 傷害與跌倒增傷。

站起完成後：

- 角色旋轉強制回正。
- 恢復正常操作。
- 觸發 `OnStandUpComplete` 掛點。

## 敵人失衡流程

### 敵人受到傷害

敵人的基礎規則比玩家簡化：

- HP 直接扣除本次傷害。
- 失衡值預設累積等同本次傷害。
- 不套用玩家用的跌倒增傷。
- 仍會套用擊退速度。

### 敵人失衡累積

敵人可累積失衡時：

- `FallTimer <= 0`
- `StaggerDamage > 0`

累積公式：

```text
StaggerValue += StaggerDamage
StaggerValue = min(StaggerValue, EnemyThreshold)
```

達到 `60` 時進入跌倒：

- `FallTimer = 1.2s`
- `StaggerValue = 0`
- 擊退速度乘 `0.3`

### 敵人失衡衰退

敵人受到失衡後會保留短暫窗口：

```text
if StaggerWindowTimer > 0:
    StaggerWindowTimer -= dt
else:
    StaggerValue -= EnemyRecoveryRate * dt
```

目前 `EnemyWindow = 1.0s`，代表敵人被打後 1 秒內失衡不會衰退，方便連續攻擊打出倒地。

### 敵人跌倒期間

`FallTimer > 0` 時：

- 不追擊。
- 不攻擊。
- 不施放特殊技能。
- 不累積新的失衡。
- 仍可受到 HP 傷害。
- 倒地角度朝 `-1.2 rad` 內插。
- 計時結束後，角色角度平滑回正並恢復 AI。

一般敵人、遠程敵人、忍者敵人與 Boss 都需遵守此規則。測試用 Dummy 可只顯示 HP / 失衡條與擊退，不要求完整 AI 停手。

## 失衡來源

### 一般傷害

大多數直接傷害會同時提供等量失衡：

- 敵人攻擊玩家：傷害值同時作為玩家失衡值。
- 玩家攻擊敵人：傷害值同時作為敵人失衡值。
- PVP 攻擊敵方三頭龍：傷害值同時作為敵方玩家失衡值。
- 地形 / DoT 傷害若走同一套 `TakeDamage`，也會提供失衡；若不希望 DoT 打跌倒，正式版需為 DoT 額外標記 `NoStagger`。

### 蓄力重擊

蓄力 Melee 需要額外提供失衡：

- 頭部蓄力重擊：額外失衡 = 本次傷害 `50%`。
- 尾巴蓄力橫掃：需提供固定高失衡，測試版對敵人使用 `160` 的額外失衡基值。
- 範圍重擊 / 震波：可傳入固定額外失衡值。

### 光束炮與組合技

原始龍光束炮：

- 每次 tick 傷害會造成等量失衡。
- 若有額外失衡倍率，需作用於每次 tick。

範圍型組合技：

- 基礎傷害造成等量失衡。
- 額外可傳入固定 `Stagger` 值，預設參考值 `70`。

## 擊退與失衡的關係

擊退與失衡同時發生，但要分成兩套資料：

- `KnockbackVelocity`：角色被推動的速度。
- `StaggerValue`：是否跌倒的累積條。

受到傷害時，若有攻擊來源位置：

```text
Direction = normalize(SelfPosition - SourcePosition)
KnockbackVelocity += Direction * KnockbackForce
```

若未指定 `KnockbackForce`，可用：

```text
KnockbackForce = FinalDamage * KnockbackBase
```

目前 `KnockbackBase = 0.2`。

進入跌倒時：

- 玩家擊退速度乘 `0.2`，跌倒期間每幀清零。
- 敵人擊退速度乘 `0.3`，之後仍可自然衰減。

## UI 顯示

HP 條下方需顯示失衡條。

- 玩家失衡條最大值：`450`。
- 敵人失衡條最大值：`60`。
- 失衡條比例：`StaggerValue / Threshold`。
- 失衡條顏色可隨比例升溫，比例越高越接近亮黃 / 橘。
- 跌倒時失衡條歸 0；不要顯示滿條卡住。

## Buff / 狀態掛點

此段為正式專案需要保留的介面，不代表本文件要重新平衡 Buff。

- `IncomingStaggerMultiplier`：玩家受到失衡倍率。
- 不屈爆發：受到失衡 `0.6x`。
- 暴走疾行：受到失衡 `1.4x`。
- 定海步：受到失衡 `0x`，等同不會跌倒。
- `OutgoingStaggerMultiplier`：攻擊方造成失衡倍率。
- 破勢重擊：造成失衡 `1.5x`。
- `RecoveryMultiplier`：玩家失衡衰退倍率。
- 不屈爆發：玩家失衡衰退 `1.2x`。
- `OnStandUpComplete`：站起完成事件。
- 不屈爆發目前會在站起完成時觸發一次範圍爆炸。
- `SuspendAllBuffs(duration)`：玩家跌倒時暫停持續 Buff，時間需覆蓋跌倒與站起。

## 狀態機建議

### 玩家三頭龍

```text
Normal
  -> StaggerAccumulating
  -> Downed
  -> StandingUp
  -> Normal
```

玩家在 `StaggerValue > 0` 但未跌倒時，仍可操作，但受到傷害會增加 `30%`。

### 敵人

```text
Active
  -> StaggerAccumulating
  -> Downed
  -> Active
```

敵人沒有站起不可操作階段；跌倒計時結束後即可恢復 AI，視覺角度平滑回正。

## 實作檢查項

- 玩家失衡條滿時，所有攻擊與集氣都必須中斷。
- 玩家跌倒與站起期間完全不可操作。
- 玩家跌倒 / 站起期間仍會受傷，且有 `30%` 增傷。
- 玩家跌倒 / 站起期間不再累積新失衡。
- 原始龍光束炮在 `PreFire` / `Firing` 被打跌倒時必須立即中斷。
- 敵人跌倒期間不能追擊、攻擊或施放特殊技能。
- 敵人受傷後有 `1.0s` 失衡保留窗口，玩家沒有保留窗口。
- 防禦加成只降低 HP 傷害，不降低玩家受到的失衡。
- 擊退速度與失衡值分開計算，不可用擊退距離代替失衡條。
- UI 失衡條應顯示目前累積值，跌倒瞬間歸 0。
