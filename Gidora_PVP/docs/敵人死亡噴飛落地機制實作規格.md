# 敵人死亡噴飛落地機制實作規格

規格件立者: lin neil  
標籤: Enemy Death / Launch Corpse / Combat Feedback  
狀態: 工程實作規格  

## 實作範圍

本文件定義敵人或 Dummy 被擊殺時，屍體 / 殘骸從原地噴飛到空中，飛行一段距離後落地的正式 Unity 實作規格。此機制屬於死亡回饋與戰場清理流程，不應混入敵人 AI、受擊失衡或掉落物邏輯。

- 適用：一般敵人、遠程敵人、忍者敵人、Boss 可破壞分身、測試用 Dummy。
- 包含：死亡瞬間處理、噴飛方向、拋物線飛行、旋轉、陰影 / 落點提示、落地衝擊、落地後事件掛點。
- 不包含：玩家三頭龍死亡、PVP 敵方三頭龍死亡、Buff 平衡、肉塊 / 掉落物吸收規則。

## 設計目標

- 讓擊殺有明確、誇張且可讀的戰鬥回饋。
- 讓屍體離開原本戰鬥熱區，避免死亡模型擋住仍在進行的命中判定。
- Dummy 被擊殺後可透過落地事件延遲重生，避免剛死亡就立即刷新造成視覺斷裂。
- 正式 Unity 實作需可調參，不可把速度、距離、時間、特效數量寫成裸數字。

## 核心流程

敵人 HP 歸 0 時，不直接刪除整個 GameObject，而是將可見模型交給死亡噴飛系統接管。

```text
Enemy.TakeDamage()
  -> HP <= 0
  -> EnemyDeathService.Kill(enemy, deathContext)
  -> 停用 AI / Collider / HealthBar / 受擊判定
  -> 建立或接管 CorpseLaunchActor
  -> CorpseLaunchActor 飛行
  -> 落地衝擊
  -> 觸發 OnCorpseLanded
  -> 清理屍體或轉成地面殘骸
```

死亡噴飛期間，原敵人不可再被鎖定、不可再受傷、不可再提供失衡值，也不可繼續執行 AI。

## 死亡瞬間處理

敵人死亡當幀需依序處理：

1. 設定 `IsDead = true`，避免重複進入死亡流程。
2. 停用 AI、尋路、攻擊、投射物發射與特殊技能。
3. 停用主碰撞器與受擊判定；若仍需要屍體碰撞，改由屍體專用碰撞層負責。
4. 銷毀或隱藏血條、失衡條、鎖定提示與地面攻擊提示。
5. 取消敵人身上的 DOT、緩速、暈眩、失衡倒地等戰鬥狀態更新。
6. 將視覺模型交給 `CorpseLaunchActor`，或生成一個死亡模型 prefab。
7. 播放死亡起飛特效與音效。

若敵人有尚未清理的子物件，例如武器、頭頂 UI、技能預警圈，需在進入噴飛前清除或從屍體模型排除。

## 噴飛方向

### 預設方向

死亡噴飛方向優先由最後一次有效傷害來源決定：

```text
BaseDirection = normalize(EnemyPosition - LastHitSourcePosition)
```

也就是敵人會往攻擊來源的反方向飛出，保留「被打飛」的直覺。

### 無來源時

若死亡來源沒有攻擊位置，例如 DOT、地形傷害、Debug kill、計時器扣血：

- 使用敵人面向的反方向。
- 若面向也不可用，使用隨機水平角度。

### 角度擾動

為避免所有屍體飛行方向過於整齊，正式版需提供水平角度擾動：

```text
LaunchDirection = RotateY(BaseDirection, RandomRange(-AngleJitter, AngleJitter))
```

擾動只作用在水平面，不改變垂直拋物線。

## 拋物線飛行

### 飛行模型

建議使用自訂 kinematic 飛行，不依賴 Unity Rigidbody 的完整物理模擬。原因是死亡噴飛需要穩定可控的距離、時間與落點，且不應被角色碰撞或臨時物件干擾到完全失控。

每幀更新：

```text
Elapsed += DeltaTime
Position += HorizontalVelocity * DeltaTime
VerticalVelocity -= Gravity * DeltaTime
Position.y += VerticalVelocity * DeltaTime
Rotation += AngularVelocity * DeltaTime
```

水平速度可套用少量阻力：

```text
HorizontalVelocity *= exp(-HorizontalDrag * DeltaTime)
```

若正式版希望嚴格控制落地距離，建議不要使用水平阻力；若使用阻力，需用預估公式或取樣修正初速。

### 以距離與時間推導速度

為了讓設計師直接調整「飛多遠」與「飛多久」，建議由距離與空中時間推導速度，而不是直接填水平速度。

```text
AirTime = RandomRange(AirTimeMin, AirTimeMax)
Distance = RandomRange(LandingDistanceMin, LandingDistanceMax)
HorizontalSpeed = Distance / AirTime
VerticalSpeed = Gravity * AirTime * 0.5
```

此公式假設起飛點與落地點高度接近。若落地高度不同，需改用實際地面高度修正：

```text
VerticalSpeed = (LandingY - StartY + 0.5 * Gravity * AirTime^2) / AirTime
```

### 高度限制

噴飛最高點需夠明顯，但不能飛到鏡頭外或造成讀取落點困難。

- 若計算出的最高點小於 `MinApexHeight`，提高 `VerticalSpeed` 或延長 `AirTime`。
- 若最高點大於 `MaxApexHeight`，縮短 `AirTime` 或降低 `VerticalSpeed`。

## 落點決定

正式版落點需在起飛時先決定，飛行期間只負責朝該落點表現拋物線。

落點計算：

```text
RawLandingPosition = StartPosition + LaunchDirection * Distance
LandingPosition = ProjectToGround(RawLandingPosition)
LandingPosition = ClampToArenaOrNavArea(LandingPosition)
```

### 地面投射

`ProjectToGround` 需以 Unity Physics 或地形系統往下 raycast，取得合法地面高度。

- 可落地層：地面、競技場、可站立平台。
- 不可落地層：水面外、牆內、不可行走障礙物內、角色碰撞器。

若落點不可用，需沿飛行方向反向縮短距離重新取樣；最多重試 `LandingResolveMaxAttempts` 次。仍失敗時，落在起飛點附近的安全位置。

### 場地邊界

若正式遊戲有有限競技場，落點必須被限制在場地內。屍體可以視覺上擦過邊界，但最終落地位置不可超出可遊玩區，以免掉落物、Dummy 重生或落地特效出現在無效區域。

## 旋轉與姿態

噴飛屍體需有明顯翻滾感。

- 起飛時給定隨機 `AngularVelocity`。
- 每個軸可有不同速度，但不要只繞 Y 軸旋轉。
- 飛行期間模型可維持原比例與材質，避免死亡瞬間視覺換模太明顯。
- 落地時可將姿態固定在最後一幀，或播放落地破碎 / 消散動畫。

若敵人使用 Animator，死亡噴飛接管後需停用 Animator root motion，避免動畫與 kinematic 位移互相覆蓋。

## 陰影與飛行可讀性

噴飛期間需要一個地面陰影或落點提示，協助玩家理解屍體高度與落地位置。

建議表現：

- 起飛瞬間播放一圈向外擴散的地面衝擊環。
- 飛行中在屍體正下方顯示 blob shadow。
- 高度越高，陰影越小且越淡。
- 落地前 `LandingWarningTime` 秒可讓落點陰影稍微放大或閃一下。

陰影只做視覺提示，不參與碰撞與傷害。

## 落地處理

屍體達到落地條件時：

```text
if Elapsed >= AirTime
or Position.y <= GroundY + GroundSnapHeight and VerticalVelocity < 0:
    Land()
```

落地瞬間需執行：

1. 將位置 snap 到地面。
2. 停止飛行更新。
3. 播放落地衝擊特效、煙塵、音效與鏡頭輕震。
4. 觸發 `OnCorpseLanded` 事件。
5. 依設定決定是否生成肉塊、掉落物或地面殘骸。
6. 在 `CorpseDespawnDelay` 後清理屍體物件。

落地衝擊預設不造成傷害，避免死亡回饋變成額外戰鬥規則。若未來要加入「大型敵人屍體砸傷」玩法，需另開傷害規格，並明確標記可傷害陣營與傷害來源。

## Dummy 特例

Dummy 使用同一套噴飛落地流程，但有額外重生掛點。

- Dummy 死亡後先噴飛，不立即重生。
- `OnCorpseLanded` 後等待 `DummyRespawnDelayAfterLanding`。
- 若 Dummy 開關仍為 ON，才重生新的 Dummy。
- 若等待期間 Dummy 開關被關閉，不重生。
- 重生位置使用正式專案既有 Dummy spawn 規則，不使用屍體落地點作為重生點。

此規則可避免玩家擊殺 Dummy 後畫面上同時出現舊屍體與新 Dummy，造成測試讀值混亂。

## 參數總表

正式 Unity 專案建議建立 `DeathLaunchConfig` ScriptableObject，或放入既有 Enemy / Combat tuning 設定檔。

| 參數 | 建議值 | 說明 |
|---|---:|---|
| `LandingDistanceMin` | `8m` | 最短落地距離 |
| `LandingDistanceMax` | `14m` | 最長落地距離 |
| `AirTimeMin` | `0.9s` | 最短空中時間 |
| `AirTimeMax` | `1.25s` | 最長空中時間 |
| `Gravity` | `30m/s^2` | 屍體飛行專用重力，不必等於世界重力 |
| `HorizontalDrag` | `0 - 1.2` | 水平阻力；若要精準距離，使用 `0` |
| `AngleJitter` | `25°` | 水平噴飛方向隨機偏移 |
| `AngularVelocityMin` | `360°/s` | 翻滾角速度下限 |
| `AngularVelocityMax` | `1080°/s` | 翻滾角速度上限 |
| `MinApexHeight` | `3.5m` | 最低拋物線高度 |
| `MaxApexHeight` | `8m` | 最高拋物線高度 |
| `GroundSnapHeight` | `0.15m` | 接近地面多少距離內可 snap |
| `LandingWarningTime` | `0.2s` | 落地前提示時間 |
| `CorpseDespawnDelay` | `0 - 1.5s` | 落地後屍體保留時間 |
| `LandingResolveMaxAttempts` | `6` | 尋找合法落點最大重試次數 |
| `DummyRespawnDelayAfterLanding` | `1.0s` | Dummy 落地後延遲重生時間 |

大型敵人可覆寫部分參數：

- 落地距離較短：避免 Boss 屍體飛出鏡頭或場地。
- 空中時間較長：凸顯重量與戲劇性。
- 落地特效尺寸較大：但仍不造成預設傷害。

## 資料結構建議

### DeathContext

死亡流程需要保留最後一擊資訊：

```csharp
public struct DeathContext
{
    public Vector3 VictimPosition;
    public Vector3? HitSourcePosition;
    public Vector3? HitDirection;
    public GameObject Killer;
    public DamageType DamageType;
    public bool IsDebugKill;
}
```

`HitSourcePosition` 優先於 `HitDirection`。若兩者都不存在，使用 fallback 方向。

### CorpseLaunchActor

`CorpseLaunchActor` 負責死亡模型飛行，不負責敵人 AI 或掉落物規則。

需要能力：

- `Initialize(GameObject visualRoot, DeathContext context, DeathLaunchConfig config)`
- `Tick(float deltaTime)`
- `Land()`
- `event Action<CorpseLaunchActor> OnLanded`
- `Dispose()` 或物件池回收

若正式專案大量生成敵人，`CorpseLaunchActor`、落地特效與陰影建議走物件池。

## 與其他系統的關係

### 敵人管理器

敵人管理器在敵人死亡後應立即從「可被 AI 更新 / 可被鎖定 / 可被命中」清單移除，但可以把死亡屍體交給獨立的 `CorpseLaunchSystem` 更新。

### 掉落物

掉落物可選擇在死亡瞬間生成或落地後生成；正式版建議落地後生成，因為比較符合「噴飛後碎開」的視覺。

若掉落物會被融合獸吸收，需確保落地後才註冊到可吸收物件清單。

### 失衡 / 擊退

死亡噴飛不使用失衡系統的 `KnockbackVelocity`。死亡噴飛是獨立狀態，進入後不再讀取敵人的受擊擊退速度。

若死亡前最後一擊有強力擊退，可只把它當作決定方向或距離倍率的參考：

```text
Distance *= DeathLaunchDistanceMultiplierFromHit
```

倍率需有上限，避免屍體飛出場地。

### PVP

本文件不定義玩家三頭龍或敵方三頭龍死亡後噴飛。PVP 勝負表現若要使用類似鏡頭或屍體飛行，需另開規格，避免與一般敵人的清場規則綁死。

## 實作檢查項

- 敵人 HP 歸 0 後只會進入一次死亡流程，不會重複生成多個屍體。
- 死亡後敵人不可再被命中、鎖定、攻擊或累積失衡。
- 屍體會往最後一擊來源的反方向噴飛；沒有來源時才使用 fallback。
- 屍體飛行有可讀的拋物線高度、水平距離與翻滾。
- 落點會被投射到合法地面，且不會超出競技場或卡進障礙物。
- 飛行期間有地面陰影或落點提示，玩家能判斷高度與落地位置。
- 落地時會 snap 到地面並播放衝擊特效。
- 落地衝擊預設不造成傷害。
- Dummy 只在屍體落地後延遲重生，且 Dummy 開關關閉時不重生。
- 所有距離、時間、速度、重力、角速度與重生延遲都能透過設定檔調整。
